How it ships

Subscription flow — step by step

User signs up & selects a plan
Supabase Auth handles login. Plan selection triggers Stripe checkout or Google/Apple in-app purchase.
Token balance created server-side
A Supabase Edge Function sets the token balance in the database on subscription confirmation. Never client-side.
Each comparison deducts one token
Atomic database transaction ensures no double-spend. User sees live token count update in real time.
Scheduled reset on billing date
Cron-triggered Edge Function resets balance at the exact billing timestamp. Tokens do not carry over — by design.
Admin panel with 2FA oversight
Control panel lets you view token balances, manually top up accounts, and see subscriber counts per tier — all secured by two-factor authentication.
